# Project Plan

## Phase 1: Requirement Analysis

Activities:

- Define problem statement.
- Identify users and use cases.
- Select technology stack.
- Define functional and non-functional requirements.

Deliverables:

- Synopsis
- SRS
- Initial architecture

## Phase 2: Design

Activities:

- Design RAG pipeline.
- Define modules.
- Design API routes.
- Design Streamlit workflow.

Deliverables:

- Architecture document
- API plan
- UI flow

## Phase 3: Implementation

Activities:

- Build document loader.
- Build text chunker.
- Integrate ChromaDB.
- Implement retrieval and answer generation.
- Build Streamlit UI.
- Build FastAPI backend.

Deliverables:

- Working source code
- Environment template
- Example files

## Phase 4: Testing

Activities:

- Unit test text preprocessing.
- Run compile checks.
- Test upload and ingestion.
- Test question answering.
- Test API endpoints.

Deliverables:

- Test results
- Manual test case table

## Phase 5: Documentation

Activities:

- Write README.
- Write setup guide.
- Write project report.
- Write presentation outline.
- Prepare viva questions.

Deliverables:

- Final documentation folder
- Presentation material outline
- Final report template

## Phase 6: Submission

Activities:

- Push final code to GitHub.
- Prepare screenshots.
- Fill student, guide, department, and university details.
- Export project report as PDF according to university format.

Deliverables:

- GitHub repository
- Project report
- Presentation slides
- Demonstration video if required
